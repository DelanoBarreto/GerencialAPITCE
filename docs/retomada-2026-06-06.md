## Decisoes importantes

- A aplicacao nao consulta a API do TCE em tempo real para montar relatorios.
- A API do TCE e usada para sincronizar dados para o Supabase/Postgres.
- Dashboards e relatorios consultam o banco local.
- O escopo operacional e `municipio + ano + grupo`.
- Filtros manuais de mes foram removidos da interface.
- Para endpoints mensais, a interface sempre considera o ano inteiro:
  - `data-inicial = AAAA01`
  - `data-final = AAAA12`
- A sincronizacao evita duplicidade por `upsert`, chaves unicas e log de sync.
- Numeros ficam como `numeric` no banco. A virgula decimal e apenas formatacao de tela.

## O que ja funciona na interface

- Mostra contexto ativo por URL, exemplo `/?municipio=014&ano=2026`.
- Permite operar por grupo: Auxiliares, BAS, ORC, BAL.
- Botoes de grupo: `Verificar` e `Sincronizar grupo`.
- A operacao por grupo roda o ano inteiro automaticamente.
- Mostra resumo de importacao por grupo.
- Mostra dashboard de execucao mensal quando ha dados.
- Quando nao ha dados importados para o ano, mostra estado vazio claro.
- Mostra logs recentes.

## Rotas internas

### `POST /api/operacao/grupo`

Exemplo:

```json
{
  "action": "check",
  "codigoMunicipio": "014",
  "ano": 2026,
  "grupo": "bal"
}
```

Roda `check:grupo` ou `sync:grupo`, sempre de janeiro a dezembro.

Teste validado:

```text
npm run check:grupo -- --municipio 014 --grupo bal --exercicio 202600 --data-inicial 202601 --data-final 202612 --default
```

### `POST /api/operacao/endpoint`

Exemplo:

```json
{
  "action": "check",
  "codigoMunicipio": "014",
  "ano": 2026,
  "endpoint": "funcoes"
}
```

Teste validado:

```text
npm run check:updates -- --municipio 014 --endpoint funcoes --exercicio 202600
```

### `POST /api/monitoramento/municipio`

Exemplo:

```json
{
  "codigoMunicipio": "014",
  "ano": 2025
}
```

Cria o monitoramento do municipio/ano e as assinaturas padrao.
